## 🔧 环境配置

### 1. Java 配置

#### 检查 Java 版本
```bash
java -version
# 输出示例: java version "17.0.11" 2024-04-16 LTS
```

#### 设置环境变量（Windows）
1. 右键「此电脑」→「属性」→「高级系统设置」→「环境变量」
2. 在「系统变量」中新建：
   - `JAVA_HOME` = `C:\Program Files\Java\jdk-17`（你的 JDK 路径）
3. 编辑 `Path`，添加：`%JAVA_HOME%\bin`

---

### 2. Maven 配置（解决 VSCode 终端无法识别 mvn）

#### 问题原因
VSCode 的 PowerShell 终端可能没有继承系统 PATH 环境变量。

#### 解决方案一：重启 VSCode（最简单）
1. **完全关闭** VSCode（包括所有窗口）
2. 重新打开 VSCode
3. 打开终端，输入 `mvn -v` 测试

#### 解决方案二：手动配置 PATH
在 VSCode 终端中临时添加：
```powershell
$env:PATH += ";C:\apache-maven-3.9.6\bin"
# 替换为你的 Maven 安装路径
```

#### 解决方案三：永久配置（推荐）
1. 打开「环境变量」→「系统变量」→「Path」
2. 添加 Maven 的 `bin` 目录路径，例如：`C:\apache-maven-3.9.6\bin`
3. **重启电脑**（必须）
4. 打开 VSCode，测试 `mvn -v`

#### 解决方案四：使用 Maven Wrapper（无需安装 Maven）
项目已配置 Maven Wrapper，直接使用：
```bash
# 在 backend 目录下
./mvnw spring-boot:run          # Linux/Mac
.\mvnw spring-boot:run           # Windows PowerShell
```

---

### 3. MySQL 配置

#### 创建数据库
```sql
CREATE DATABASE exam_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE exam_db;
```

#### 导入初始化脚本
```bash
mysql -u root -p exam_db < backend/src/main/resources/schema.sql
# 输入密码：Lm147187
```

#### 修改数据库连接配置
编辑 `backend/src/main/resources/application.yml`：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/exam_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: Lm147187  # 你的数据库密码
```

## 🚀 启动步骤

### 方法一：命令行启动（推荐）

#### 1. 启动后端

```bash
# 进入后端目录
cd d:\kaoshisystem\backend

# 方法A：使用系统 Maven（需配置环境变量）
mvn spring-boot:run

# 方法B：使用 Maven Wrapper（无需配置环境变量）
.\mvnw spring-boot:run

# 方法C：编译后运行
mvn clean package
java -jar target/exam-system-1.0.0.jar
```

后端启动成功标志：
```
Started ExamApplication in X.XX seconds (process XXXX)
```

#### 2. 启动前端

```bash
# 进入前端目录
cd d:\kaoshisystem\frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

前端启动成功标志：
```
VITE v5.x.x  ready in XXXX ms
➜  Local:   http://localhost:5173/
```

### 方法二：VSCode 界面启动

#### 后端启动
1. 打开 `backend/src/main/java/com/kaoshi/exam/ExamApplication.java`
2. 点击编辑器右上角的 **▶️ Run** 按钮
3. 或按 `Ctrl+F5` 运行

#### 前端启动
1. 打开 `frontend/package.json`
2. 在「Scripts」中找到 `dev`
3. 点击右侧的 **▶️ Run** 按钮

### 方法三：Spring Boot Dashboard

1. 确保安装了 **Spring Boot Extension Pack** 扩展
2. 左侧边栏点击 **Spring Boot Dashboard**
3. 在 `APPS` 下找到 `exam-system`
4. 点击播放按钮 ▶️

## ❌ 常见问题

### Q1: VSCode 终端无法识别 mvn？
```
mvn : 无法将“mvn”项识别为 cmdlet、函数、脚本文件或可运行程序的名称。
```
**解决方案**：
1. 检查 Maven 是否已添加到系统 PATH
2. **重启 VSCode**（关键！）
3. 或使用 `.\mvnw spring-boot:run`

### Q2: 数据库连接失败？
```
Error creating bean with name 'dataSource'
```
**解决方案**：
1. 确认 MySQL 服务已启动：`net start mysql`
2. 确认密码正确（已配置为 `Lm147187`）
3. 确认数据库 `exam_db` 已创建

### Q3: 前端启动报错？
```
npm : 无法加载文件 ... npm.ps1
```
**解决方案**：
```powershell
# 临时允许脚本执行
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q4: Maven 下载依赖慢？
**解决方案**：配置阿里云镜像，在 `~/.m2/settings.xml` 中添加：
```xml
<mirrors>
    <mirror>
        <id>aliyunmaven</id>
        <mirrorOf>central</mirrorOf>
        <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
</mirrors>
```

### Q5: Redis 连接失败？
**解决方案**：
1. 安装 Redis 并启动：`redis-server`
2. 或暂时注释 `application.yml` 中的 Redis 配置（部分功能不可用）
